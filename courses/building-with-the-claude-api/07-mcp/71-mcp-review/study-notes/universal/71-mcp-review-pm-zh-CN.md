## 一句话摘要

MCP 的三个建构模组 — tools、resources 和 prompts — 各服务产品中的不同利害关係人：AI 模型、应用程序本身或终端使用者，知道每个模组服务誰是做出正确架构选择的关键。

---

## 利害关係人对应表

把建构产品想像成有三个部门，各自需要不同的東西：

| 建构模组 | 利害关係人 | 商業类比 | 何时使用 |
|----------|-----------|----------|---------|
| **Tools** | AI 模型 | 賦予員工新技能 | Claude 需要做新事情（计算、搜尋、查询） |
| **Resources** | 应用程序 | 访问公司数据库 | 你的应用程序需要数据來显示或作为上下文 |
| **Prompts** | 终端使用者 | 预建报表模板 | 使用者需要一键访问常用工作流程 |

### Tools = AI 的新技能

当产品需要 Claude 做文字生成以外的事情时 — 执行代码、搜尋数据库、调用外部 API — 你提供工具。关键区別：**Claude 决定何时使用它**，不是你的代码也不是使用者。

**产品决策**：「我们的 AI 需要新能力嗎？」→ 新增工具。

### Resources = 按需数据

当应用代码需要数据來显示在 UI 中或丰富提示词时，使用 resource。应用程序拉取数据 — 就像产品订阅的数据源。

**产品决策**：「我们的应用程序需要拉取外部数据嗎？」→ 新增 resource。

### Prompts = 使用者工作流程

当使用者需要一键触发的预建、最佳化工作流程时，建立 prompts。把它们想成使用者啟动的模板。

**产品决策**：「使用者需要常見任务的捷徑嗎？」→ 新增 prompt。

---

## PRD 决策清单

在 PRD 中指定新的 MCP 整合时：

- [ ] **誰啟动操作？** AI → Tool、应用程序 → Resource、使用者 → Prompt
- [ ] **结果是什麼？** 新能力 → Tool、数据提取 → Resource、工作流程 → Prompt
- [ ] **是重复性还是一次性？** 考慮哪个基本元素最易於维护

## 记忆卡

| 问题 | 答案 |
|------|------|
| MCP 服务器的三大基本元素是什麼？ | Tools、Resources 和 Prompts。 |
| MCP tools 服务哪个利害关係人？ | AI 模型 — Claude 决定何时调用工具。 |
| MCP resources 服务哪个利害关係人？ | 应用程序 — 应用代码决定何时取得数据。 |
| MCP prompts 服务哪个利害关係人？ | 终端使用者 — 使用者透过 UI 或命令触发 prompts。 |
| 如果产品需要 Claude 搜尋数据库，该用哪个基本元素？ | Tool — 它新增 Claude 可以决定使用的新能力。 |
| 如果 UI 需要显示文件列表，该用哪个基本元素？ | Resource — 你的应用代码取得数据來显示。 |
| 如果使用者需要一键「生成报告」按鈕，该用哪个基本元素？ | Prompt — 它是使用者触发的预定义工作流程。 |
