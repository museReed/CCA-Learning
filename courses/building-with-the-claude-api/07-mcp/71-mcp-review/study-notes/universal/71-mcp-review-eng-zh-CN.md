## 一句话摘要

MCP 的三个服务器基本元素 — tools、resources 和 prompts — 各由应用程序的不同部分控制：**tools** 由模型控制、**resources** 由应用程序控制、**prompts** 由使用者控制，这个对应关係告訴你在新增功能时该选择哪个元素。

---

## 三大基本元素与其控制者

| 基本元素 | 控制者 | 用途 | 范例 |
|-----------|--------|------|------|
| **Tools** | 模型 (Claude) | 扩展 Claude 的能力 | Claude 决定执行 JavaScript 來计算 √3 |
| **Resources** | 应用代码 | 为应用程序/UI 提供数据 | 从 Google Drive 取得文件列表用於自动完成或上下文增強 |
| **Prompts** | 使用者 | 预定义工作流程 | 使用者点击按鈕或输入斜线命令啟动工作流程 |

### Tools — 模型控制

Claude 独自决定何时调用工具。你的代码定义工具及其 schema，但模型在执行时做出决定。如果你需要**为 Claude 新增能力**（执行代码、查询数据库、调用 API），就在 MCP 服务器上实作工具，或透过 MCP 客户端使用现有服务器的工具。

### Resources — 应用程序控制

应用代码决定何时取得资源。资源提供数据 — 它们不执行操作。课程项目中的兩个常見模式：

1. **UI 丰富化** — 取得资源來填充自动完成建议
2. **提示词增強** — 取得资源來在发送给 Claude 之前注入上下文

如果你需要**将数据放入应用程序**以供显示或提供上下文，使用 resource。

### Prompts — 使用者控制

使用者透过点击 UI 元素或输入斜线命令來啟动 prompt。Prompts 是预定义、已最佳化的工作流程 — 想想 Claude.ai 聊天输入框下方的按鈕列。每个按鈕都触发一个预先撰写的提示模板。

如果你需要**实作使用者啟动的预定义工作流程**，使用 prompt。

---

## 决策框架

```
需要为 Claude 新增能力？          → Tool
需要将数据放入应用程序/UI？       → Resource
需要使用者触发的预定义流程？      → Prompt
```

这些是高层指导原则，不是硬性规则。关键洞察是每个基本元素服务不同的**受眾**：模型、应用程序或使用者。

---

## CCA 考试关聯

- **D2 (MCP 整合)**：预期会出现「你应该在什麼情況下使用哪个 MCP 基本元素？」类型的题目 — 答案直接对应到誰控制该操作（模型/应用程序/使用者）。
- 注意：「Tools 由模型控制」是最常考的事实。Resources 和 prompts 常被混淆 — 记住 resources 是应用代码用來取得数据的，prompts 是使用者啟动的工作流程。

## 记忆卡

| 问题 | 答案 |
|------|------|
| MCP 服务器的三大基本元素是什麼？ | Tools、Resources 和 Prompts。 |
| 誰在执行时控制 MCP tools？ | 模型 (Claude) — 它决定何时调用工具。 |
| 誰在执行时控制 MCP resources？ | 应用代码 — 你的应用程序决定何时取得数据。 |
| 誰在执行时控制 MCP prompts？ | 使用者 — 他们透过 UI 元素或斜线命令触发 prompts。 |
| 何时该用 tool，何时该用 resource？ | Tool：为 Claude 新增能力。Resource：将数据放入应用程序/UI。 |
| Claude.ai 中 resource 的范例是什麼？ | 「从 Google Drive 新增」按鈕 — 应用程序取得並显示文件列表。 |
| Claude.ai 中 prompt 的范例是什麼？ | 聊天输入框下方的预建对话起始按鈕。 |
| 这些基本元素-控制者的对应是硬性规则嗎？ | 不是 — 它们是高层指导原则，指示每个基本元素的主要目标受眾。 |
