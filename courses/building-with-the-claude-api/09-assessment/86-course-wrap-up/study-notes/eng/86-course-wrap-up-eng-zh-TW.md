## 一句話摘要

本課總結回顧了課程涵蓋的所有主要主題 — 從模型選擇和 API 參數、到提示詞評估、工具使用、MCP 和代理架構 — 並指出你應該獨立研究的後續主題：代理編排、代理評估、Agentic RAG 和工具評估。

## 課程回顧

### 模型與 API 存取 (D5)

Anthropic 提供多種模型，各有不同的取捨：
- **Haiku** — 快速、低成本，最適合較小的請求
- **Sonnet** — 更高智能，最適合複雜任務

API 接受影響 Claude 行為的參數：**temperature**（創造力控制）、**stop sequences**（輸出終止）和 **message prefilling**（引導回應格式）。

### 提示詞評估 (D5)

講師強調這是**生產應用最重要的單一實踐**。在本地執行提示詞 10 次可能看起來沒問題，但生產環境的使用者會發現邊緣案例。關鍵原則：

- 評估不需要複雜的框架 — Claude 可以生成評估框架
- 系統化評估能抓到手動測試遺漏的失敗
- 課程本身的評估框架大部分由 Claude 撰寫

### 提示工程 (D4, D5)

核心技術：**清晰且直接**。明確告訴 Claude 你的期望。課程中涵蓋的其他技術都建立在這個基礎上，但清晰度始終是最主要的槓桿。

### 工具使用 (D2)

工具使用大幅擴展了 Claude 的能力。這是課程中最複雜的部分之一，因為涉及工具 schema 定義、處理訊息區塊（tool_use、tool_result）、多輪工具對話和同時使用多個工具。

### 工作流程 vs 代理 (D1)

關鍵區別：**工作流程通常比代理提供更好的結果和更高的準確度**。代理很令人興奮，但對於結構化、可預測的任務，工作流程應該是你的預設選擇。

## 建議後續研究主題

| 主題 | 重要性 |
|------|--------|
| **代理編排** | 讓多個代理協同工作 |
| **代理評估與監控** | 在生產環境中衡量代理效能 |
| **Agentic RAG** | 代理主動搜尋和推理的 RAG 變體 |
| **RAG 評估** | 系統化衡量檢索品質 |
| **工具評估** | 確保工具描述幫助 Claude 正確使用工具（類似提示詞評估） |

---

## CCA 考試關聯

本課是後設複習 — 將整個課程對應到考試領域。用它作為檢查清單：
- D1：代理 vs 工作流程的區別、agentic loop
- D2：工具使用、MCP 基本元素
- D4：提示工程、安全性
- D5：模型選擇、提示詞評估、生產模式
