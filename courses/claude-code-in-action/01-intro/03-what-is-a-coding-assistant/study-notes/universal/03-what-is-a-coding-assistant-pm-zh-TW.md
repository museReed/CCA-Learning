## 重點摘要

Coding assistant 不只是回答程式問題的 AI 聊天機器人。它是一個能自主蒐集資訊、制定計畫、並透過「tool use」機制執行真實動作（讀檔、寫程式碼、跑指令）的自主系統。PM 需要理解這點，因為它決定了 AI 程式產品能做什麼、不能做什麼，以及為什麼 Claude Code 的架構具有競爭優勢。

---

## 為什麼 PM 需要理解這些

身為 PM，你不需要建造 coding assistant，但必須理解：

1. **Coding assistant 與 chatbot 的差別** — 才能對利害關係人設定正確期望
2. **Tool use 如何運作** — 才能評估 AI 產品的能力與限制
3. **為什麼不是所有 AI 模型都一樣** — 才能做出有根據的 build-vs-buy 決策

---

## 心智模型：新人報到類比

把 coding assistant 想成新進資深工程師的 onboarding 過程：

| 階段 | 新人 | Coding Assistant |
|------|------|-----------------|
| **蒐集上下文** | 閱讀文件、探索程式碼庫、問問題 | 讀取檔案、搜尋程式碼、檢視專案結構 |
| **規劃** | 把任務拆成子任務、找出相依性 | 決定要讀哪些檔案、以什麼順序工作 |
| **執行動作** | 寫程式碼、跑測試、commit 變更 | 編輯檔案、執行指令、建立新檔案 |
| **評估** | 自我審查、再跑一次測試 | 檢查結果，如果失敗就回到前面的步驟 |

與 chatbot 的關鍵差異：chatbot 像在走廊遇到新人問個問題，他給你一個回答就走了。Coding assistant 像指派一張 ticket 給他 — 他會持續工作直到完成。

## Tool Use 問題（與解決方案）


![Tool Use Bridge Mechanism](../../visuals/tool-use-bridge-mechanism-zh-TW.svg)
*圖：工具使用如何連接文本生成與真實世界動作。*

PM 必須理解的根本限制：

**AI language model 只能讀寫文字。** 它不能開檔案、跑程式或修改程式碼。它就像一個坐在沒有電腦的密室裡的天才顧問 — 能思考和建議，但什麼都做不了。

**Tool use 就是解決方案。** Coding assistant 扮演中介者：

1. AI 說：「我需要讀取主設定檔」
2. 助手把這翻譯成實際的檔案讀取動作
3. 檔案內容被呈現給 AI
4. AI 現在能做出有根據的決策

這就像安全設施裡的收發室系統：顧問在表單上寫下請求，收發室取回文件送回。顧問從不離開房間，但仍能取得需要的所有資料。

> 🎬 **講師影片觀點**
>
> 講師解釋 coding assistant 會「把指令加入 context」讓 AI 知道如何請求工具。AI 不是天生就知道怎麼要求檔案或跑指令 — 它從助手提供的指令中學習。

## 為什麼 Claude 的優勢對產品決策重要

不是所有 AI 模型都能同樣好地處理 tool use。Claude（Opus、Sonnet、Haiku）在這方面表現出色。PM 該在意的原因：

| 商業影響 | 意義 |
|---------|------|
| **處理複雜工作流程** | Claude 能可靠地串聯 10+ 次工具呼叫不迷失。較弱的模型在 3-4 次就失敗。這直接影響你能自動化哪些任務。 |
| **容易擴充** | 新增功能（資料庫查詢、API 呼叫、自訂工具）能可靠運作，因為 Claude 擅長理解工具描述。降低整合成本。 |
| **程式碼不外洩** | Claude Code 按需讀取檔案而非預先索引。你的程式碼庫不會儲存在外部。簡化安全審查與合規。 |

> 💡 **PM 決策框架**
>
> 評估 AI 程式工具時，問：「它如何存取和修改程式碼？」如果答案涉及上傳或索引整個程式碼庫，那是與按需 tool use 不同（且風險更高）的架構。

## 產品情境推演

### 情境：為工程團隊評估 AI 工具

CTO 請你比較三個 AI 程式工具。運用你學到的知識：

| 要問的問題 | 該看什麼 | 警訊 |
|-----------|---------|------|
| 它如何存取我們的程式碼？ | 透過工具按需讀取檔案 | 需要上傳整個程式碼庫到外部伺服器 |
| 能處理多步驟任務嗎？ | 展示可靠的 agentic loop（上下文 → 規劃 → 執行 → 評估） | 只能做單輪問答 |
| 能加自訂工具嗎？ | 可擴充的工具系統，介面清楚 | 固定功能集，無法客製化 |
| 安全模型是什麼？ | 不預先索引，只讀需要的檔案 | 預先索引所有內容，embedding 存在外部 |

## 練習題

### 問題 1：利害關係人溝通

CEO 問：「為什麼不直接用 ChatGPT 寫程式？它回答程式問題看起來很厲害。」根據本課，最準確的回答是什麼？

- A. ChatGPT 完全無法理解程式碼
- B. 回答程式問題與自主處理程式碼是不同的事；coding assistant 需要強大的 tool use 才能在 agentic loop 中讀檔、寫程式碼和執行指令
- C. ChatGPT 較便宜但 Claude Code 較快
- D. Claude Code 有更多程式相關的訓練資料

<details><summary>答案與解析</summary>

**B** — 核心課程重點是 chatbot（文字問答）與 coding assistant（agentic loop + tool use）根本不同。差異不在知識量，而在透過工具執行真實動作的能力。

- A 事實上錯誤 — ChatGPT 能理解程式碼
- C 和 D 不是本課教的內容

**PM 重點**：向非技術利害關係人解釋 AI 程式工具時，聚焦在執行動作的能力（tool use + agentic loop），而非模型的「聰明程度」。
</details>

### 問題 2：安全審查情境

安全團隊對採用 Claude Code 有疑慮，問：「Anthropic 會儲存我們程式碼庫的副本嗎？」根據本課，正確的回答是什麼？

- A. 是的，Claude Code 會索引整個程式碼庫以加速搜尋
- B. 不會，Claude Code 透過 tool use 按需讀取檔案 — 不需要預先索引或外部儲存
- C. 只有 Claude 讀過的檔案會暫存在 Anthropic 的伺服器
- D. 程式碼庫儲存在本地加密的向量資料庫中

<details><summary>答案與解析</summary>

**B** — 課程明確指出 Claude Code 基於工具的架構不需要預先索引。檔案在需要時按需讀取。

- A 描述的是某些競品的架構，不是 Claude Code
- C 加入了課程未涵蓋的細節
- D 描述的是 RAG 方法，不是 Claude Code 的架構

**PM 重點**：Claude Code 的按需讀取模式是重大安全優勢。在合規討論和安全審查中善用這點。
</details>

### 問題 3：供應商評估

你正在評估兩個 AI 程式工具。工具 A 預先索引整個程式碼庫，提供即時搜尋。工具 B 透過 tool use 按需讀取檔案。哪個取捨分析正確？

- A. 工具 A 永遠較好，因為即時搜尋意味著更快的結果
- B. 工具 B 永遠較好，因為按需讀取更安全
- C. 工具 A 以安全換取速度（程式碼儲存在外部）；工具 B 以初始速度換取安全（無外部儲存）— 正確選擇取決於你組織的風險容忍度
- D. 兩種方法之間沒有有意義的差別

<details><summary>答案與解析</summary>

**C** — 這是比例適當的回應。兩種架構都有取捨。課程強調按需讀取是安全優勢，但並未宣稱它在所有方面都優越。

- A 忽略了安全影響
- B 過於絕對 — 索引可能有合理的使用場景
- D 忽略了根本的架構差異

**PM 重點**：AI 工具的產品決策應權衡安全需求與效能需求。理解底層架構（tool use vs. 索引）讓你能明確而非不知不覺地做出這個取捨。
</details>
