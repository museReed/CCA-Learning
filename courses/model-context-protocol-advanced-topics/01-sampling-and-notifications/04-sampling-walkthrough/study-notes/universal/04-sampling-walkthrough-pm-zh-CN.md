## 一句话摘要

Sampling 是让 MCP 服务器代替自己向客户端请求 AI 文本生成的机制——把它想成「向总部汇报」，外勤人员（服务器）在任务中向指挥中心（客户端）请求情报。

---

## PM 为何需要了解 Sampling

Sampling 改变了 MCP 的一般流程。通常是客户端要求服务器做事（执行工具、读取资源）。有了 Sampling，方向反转：服务器要求客户端生成文本。这对产品设计很重要，因为：

1. **服务器端智能** — 你的服务器工具可以在执行过程中融入 AI 推理
2. **客户端控制 LLM** — 客户端决定使用哪个模型以及如何调用
3. **人机协作潜力** — 客户端可以在生成文本之前插入人工审核步骤

---

## 心智模型：餐厅厨房

| 角色 | MCP 对应 | 发生的事 |
|------|---------|---------|
| 厨房（服务器） | MCP 服务器工具 | 发现需要一个餐酒搭配建议 |
| 服务员跑回去找侍酒师 | Sampling 请求 | 服务器发送消息请求文本生成 |
| 侍酒师（客户端 + LLM） | 客户端的 sampling 回调 | 运用专业知识生成红酒建议 |
| 服务员带着答案回来 | `CreateMessageResult` | 服务器收到生成的文本并继续工作 |

关键洞察：厨房（服务器）永远不会直接和侍酒师（LLM）对话。它一定是通过服务员（客户端回调）。

---

## 六步流程

1. **服务器需要 AI 文本** — 在工具调用期间，服务器决定它需要 LLM 生成的内容
2. **服务器发送请求** — 提供描述需求的消息
3. **客户端接收请求** — 其预先注册的回调函数启动
4. **客户端生成文本** — 使用它偏好的任何 AI 模型和 SDK
5. **客户端返回结果** — 格式化为标准 MCP 结果
6. **服务器继续工作** — 使用生成的文本完成其任务

---

## 产品设计影响

| 考量面向 | 影响 |
|---------|------|
| **模型选择权在客户端** | 你的服务器不需要知道使用哪个 LLM——它只需要请求文本 |
| **成本控制** | 客户端可以强制限制 token 数量、选择较便宜的模型或缓存响应 |
| **安全性** | API 密钥保留在客户端；服务器永远看不到 |
| **多步骤工作流程** | 单次工具调用可以发起多次 sampling 请求，实现复杂推理链 |
| **需要格式转换** | MCP 消息格式与 LLM SDK 格式不同——工程团队必须处理这个转换 |

---

## 何时在产品中使用 Sampling

| 场景 | 为何用 Sampling | 替代方案 |
|------|----------------|---------|
| 服务器工具需要摘要检索的数据 | 服务器可以在工具执行中请求摘要 | 返回原始数据让客户端摘要（较不优雅） |
| 多智能体编排 | 子智能体（服务器）可以向主智能体（客户端）请求推理 | 将所有逻辑构建在一个庞大的客户端中 |
| 内容生成管线 | 服务器处理工作流程步骤，通过客户端委派写作给 LLM | 硬编码模板替代 AI 生成 |

## 常见误解

| 误解 | 事实 |
|------|------|
| 「服务器直接调用 LLM」 | 不是——服务器向客户端请求，由客户端调用 LLM |
| 「Sampling 使用与 Anthropic API 相同的消息格式」 | 不是——MCP 有自己的消息类型，需要转换 |
| 「服务器选择模型」 | 不是——客户端的回调函数决定模型 |
| 「Sampling 是自动的」 | 不是——客户端必须明确注册回调函数 |

- 考试测试对客户端控制模型选择和 API 密钥的理解
- 考题可能呈现 sampling 缺失的情境，要求你找出根本原因（缺少回调注册）

## 记忆卡

| # | 问题 | 答案 |
|---|------|------|
| 1 | 在 MCP sampling 中，哪个组件发起文本生成请求？ | 服务器在工具执行期间发起 |
| 2 | 在 sampling 中，哪个组件实际调用 LLM？ | 客户端，通过其 sampling 回调函数 |
| 3 | 服务器知道客户端使用哪个 LLM 模型吗？ | 不知道——模型选择完全是客户端的责任 |
| 4 | 为什么 sampling 期间需要格式转换？ | MCP 使用自己的消息类型，与 LLM SDK 格式（如 Anthropic）不同 |
| 5 | LLM 的 API 密钥在 sampling 流程中位于哪里？ | 仅在客户端——服务器永远看不到 |
| 6 | 如果客户端未注册 sampling 回调会怎样？ | 服务器的 sampling 请求将会失败 |
| 7 | 服务器能否在一次工具执行中发起多次 sampling 调用？ | 可以——它可以为复杂的工作流程串联多次请求 |
| 8 | Sampling 如何影响成本控制？ | 客户端控制 token 限制、模型选择和缓存，拥有完整的成本控制权 |
