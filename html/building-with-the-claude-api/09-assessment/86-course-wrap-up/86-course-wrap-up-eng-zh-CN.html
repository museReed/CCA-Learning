一句话摘要

本课总结回顧了课程涵蓋的所有主要主题 — 从模型选择和 API 参数、到提示词评估、工具使用、MCP 和代理架构 — 並指出你应该独立研究的后续主题：代理编排、代理评估、Agentic RAG 和工具评估。

课程回顧

模型与 API 访问 (D5)

Anthropic 提供多种模型，各有不同的取捨：
- Haiku — 快速、低成本，最适合較小的请求
- Sonnet — 更高智能，最适合复杂任务

API 接受影响 Claude 行为的参数：temperature（创造力控制）、stop sequences（输出终止）和 message prefilling（引导回应格式）。

提示词评估 (D5)

讲師強调这是生产应用最重要的单一实踐。在本地执行提示词 10 次可能看起來没问题，但生产环境的使用者会发现边緣案例。关键原则：

  • 评估不需要复杂的框架 — Claude 可以生成评估框架
  • 系统化评估能抓到手动测试遗漏的失敗
  • 课程本身的评估框架大部分由 Claude 撰写

提示工程 (D4, D5)

核心技术：清晰且直接。明确告訴 Claude 你的期望。课程中涵蓋的其他技术都建立在这个基础上，但清晰度始终是最主要的槓桿。

工具使用 (D2)

工具使用大幅扩展了 Claude 的能力。这是课程中最复杂的部分之一，因为涉及工具 schema 定义、处理信息区块（tool_use、tool_result）、多轮工具对话和同时使用多个工具。

工作流程 vs 代理 (D1)

关键区別：工作流程通常比代理提供更好的结果和更高的准确度。代理很令人興奮，但对於结构化、可预测的任务，工作流程应该是你的默认选择。

建议后续研究主题

主题 重要性
代理编排 让多个代理协同工作
代理评估与监控 在生产环境中衡量代理效能
Agentic RAG 代理主动搜尋和推理的 RAG 变体
RAG 评估 系统化衡量检索品质
工具评估 确保工具描述幫助 Claude 正确使用工具（类似提示词评估）

CCA 考试关聯

本课是后设复習 — 将整个课程对应到考试领域。用它作为检查清单：
- D1：代理 vs 工作流程的区別、agentic loop
- D2：工具使用、MCP 基本元素
- D4：提示工程、安全性
- D5：模型选择、提示词评估、生产模式

记忆卡

问题 答案
哪个模型最适合快速、低成本的请求？ Haiku。
讲師认为生产 AI 应用最重要的实踐是什麼？ 提示词评估。
最主要的提示工程技术是什麼？ 清晰且直接 — 明确告訴 Claude 你的期望。
对於结构化任务，应该默认使用代理还是工作流程？ 工作流程 — 通常提供更好的结果和更高的准确度。
什麼是工具评估？ 确保工具描述幫助 Claude 正确使用工具 — 类似提示词评估。
什麼是 Agentic RAG？ 代理主动搜尋和推理的 RAG 变体，而非简单的检索-生成。
讲師建议研究哪些后续主题？ 代理编排、代理评估/监控、Agentic RAG、RAG 评估、工具评估。
