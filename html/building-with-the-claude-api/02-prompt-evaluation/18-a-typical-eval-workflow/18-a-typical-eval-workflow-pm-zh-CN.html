一句话摘要

Prompt eval 工作流是 AI 团队版的"产品上线 checklist" — 可重复的五步流程，产出一个 PM 能写进发布文档、OKR、优先级决策的单一质量数字。

心智模型：食谱试验厨房

专业试验厨房不会靠一次试吃就决定是否推出新款早餐麦片。他们有固定的试吃小组、固定的评分表，然后对每一个版本重复打分直到平均分跨过门槛。这就是 prompt evaluation 对 prompt 做的事。

试验厨房 Prompt eval
初版食谱 起草的 prompt
试吃小组 Eval 数据集
试吃员品尝每一匙 喂进 Claude
试吃员按评分表打 1-10 分 喂进 grader
主厨微调食谱、重新评分 迭代 prompt 并重复

数据集就是小组。Grader 就是评分表。Prompt 就是食谱。小组和评分表固定，只改食谱 — 这就是隔离"有效改动"的方法。

五个步骤 — PM 速查表

步骤 产出 PM 价值
1. 起草 prompt Baseline 版本 可度量、有可改进的基准
2. 建立数据集 N 条真实输入 定义"质量"在此功能的含义
3. 喂进 Claude N 条原始输出 当前行为的证据
4. 喂进 grader 数值分数（每条 1-10，取平均） 能写进发布文档的质量指标
5. 迭代 新 prompt + 新分数 证明团队真的在改进产品

Grader 吐出来的数字，就是你跟财务、市场、高层争论时的武器。没有第 4 步就没有数字，没有数字就没法争，没法争就没法排优先级。

课程实例（为什么这关键）

课程用三个问题的玩具数据集，示范改 prompt 一行让平均分从 7.66 升到 8.7。整个要点就浓缩为这件事：

  • 改动前："这更好吗？"→ 意见。
  • 改动后："平均分从 7.66 升到 8.7。"→ 事实。

你现在可以用一个真实 delta 更新发布文档、OKR 仪表盘、工程 roadmap。那个数字让 prompt 工作对组织里其他人"可见、可读"。

场景 注意事项

PM 决策框架

当工程团队说新版 prompt 好了，走过这四个问题：

问题 好答案长什么样
当前 prompt 的 baseline 分数？ 来自固定数据集的 1-10 分数字
提案 prompt 的新分数？ 来自同一数据集、同一 grader 的数字
数据集跟上次跑相比有变过吗？ 没 — 若有，比较无效
提升幅度值得上线吗？ 主观，但至少大家在讨论的是数字

任何一个问题的答案是"我们没做"，你就该挡下来。团队现在要你上线的是意见，不是度量过的改进。

PM 常见错误

  1. 接受"我们测过更好" — 永远要问 baseline 数字和新数字。
  2. 让团队在两次跑之间悄悄改数据集 — 这会摧毁可比性，让所有声称的改进失效。
  3. 把 grader 当成"技术细节、PM 不用管" — Grader 就是你的质量定义；不拥有它等于放弃产品判断权。
  4. 分数升一点点就庆祝"已上线" — 第一个改进很少是最佳的；要求持续迭代直到分数进入平台期。
  5. 忘记把分数写进发布文档 — 没写下来，组织会忘记"好"的定义是什么。

Key Insight

五步工作流把 prompt engineering 从"相信我们就好"变成"看这个数字"。CCA 考试里只要 D3 问到流程、可复现性、迭代，答案就是把五个步骤按顺序讲一遍，并强调数据集保持不变。对 PM 而言，这是让你不必变成 prompt engineer，也能让工程团队对 AI 质量负责的机制。

CCA 考试相关性

  • D3（Evaluation & Iteration）：五步循环是 THE canonical workflow。考题可能把步骤打乱要你排序。
  • D5（Enterprise Deployment）：这条循环是"工程觉得好了"与"上线"之间的闸门。
  • 注意"实验中途换 dataset"的情境 — 正确答案永远是数据集必须保持不变。

Flashcards

Front Back
典型 prompt eval 工作流的五个步骤顺序是？ 起草 prompt → 建立数据集 → 喂进 Claude → 喂进 grader → 迭代。
Grader 产出什么？ 通常是 1-10 分的数值分数，10 代表完美答案。
为什么数据集在迭代之间必须保持不变？ 这样分数差异才能归因于 prompt 改动，而不是输入改动。
课程示范的具体 delta 是多少？ 加上 "Answer the question with ample detail" 之后，平均分从 7.66 升到 8.7。
PM 看数据集的试验厨房类比？ 试吃小组 — 保持固定，让分数变化只反映食谱（prompt）的变化。
PM 批准新版 prompt 前应该问什么？ Baseline 分数、新分数、数据集未变、以及提升是否值得上线。
两次跑之间改数据集的风险？ 可比性被摧毁 — 无法判断分数变动来自 prompt 还是输入。
哪个 CCA domain 最直接测这个工作流？ D3 Evaluation & Iteration — 五步顺序是本章最可测的概念。
