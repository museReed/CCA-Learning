MCP 複習 — PM 快速掃描

項目 細節
考試領域 D2 — 工具設計與 MCP 整合 (18%)
任務陳述 2.1 (MCP 架構)、2.2 (基本元素)、2.3 (tool vs resource vs prompt)
來源 building-with-the-claude-api / 07-mcp / 第 71 課

一句話摘要

MCP 的三個建構模組 — tools、resources 和 prompts — 各服務產品中的不同利害關係人：AI 模型、應用程式本身或終端使用者，知道每個模組服務誰是做出正確架構選擇的關鍵。

利害關係人對應表

把建構產品想像成有三個部門，各自需要不同的東西：

建構模組 利害關係人 商業類比 何時使用
Tools AI 模型 賦予員工新技能 Claude 需要做新事情（計算、搜尋、查詢）
Resources 應用程式 存取公司資料庫 你的應用程式需要資料來顯示或作為上下文
Prompts 終端使用者 預建報表模板 使用者需要一鍵存取常用工作流程

Tools = AI 的新技能

當產品需要 Claude 做文字生成以外的事情時 — 執行程式碼、搜尋資料庫、呼叫外部 API — 你提供工具。關鍵區別：Claude 決定何時使用它，不是你的程式碼也不是使用者。

產品決策：「我們的 AI 需要新能力嗎？」→ 新增工具。

Resources = 按需資料

當應用程式碼需要資料來顯示在 UI 中或豐富提示詞時，使用 resource。應用程式拉取資料 — 就像產品訂閱的資料源。

產品決策：「我們的應用程式需要拉取外部資料嗎？」→ 新增 resource。

Prompts = 使用者工作流程

當使用者需要一鍵觸發的預建、最佳化工作流程時，建立 prompts。把它們想成使用者啟動的模板。

產品決策：「使用者需要常見任務的捷徑嗎？」→ 新增 prompt。

PRD 決策清單

在 PRD 中指定新的 MCP 整合時：

  • [ ] 誰啟動操作？ AI → Tool、應用程式 → Resource、使用者 → Prompt
  • [ ] 結果是什麼？ 新能力 → Tool、資料擷取 → Resource、工作流程 → Prompt
  • [ ] 是重複性還是一次性？ 考慮哪個基本元素最易於維護

CCA 考試關聯

  • D2 (MCP 整合)：tool/resource/prompt 決策框架是重點考試內容。記住控制者對應關係。
  • 關鍵公式：Tools = 模型控制、Resources = 應用程式控制、Prompts = 使用者控制

記憶卡

問題 答案
MCP 伺服器的三大基本元素是什麼？ Tools、Resources 和 Prompts。
MCP tools 服務哪個利害關係人？ AI 模型 — Claude 決定何時呼叫工具。
MCP resources 服務哪個利害關係人？ 應用程式 — 應用程式碼決定何時取得資料。
MCP prompts 服務哪個利害關係人？ 終端使用者 — 使用者透過 UI 或指令觸發 prompts。
如果產品需要 Claude 搜尋資料庫，該用哪個基本元素？ Tool — 它新增 Claude 可以決定使用的新能力。
如果 UI 需要顯示文件列表，該用哪個基本元素？ Resource — 你的應用程式碼取得資料來顯示。
如果使用者需要一鍵「生成報告」按鈕，該用哪個基本元素？ Prompt — 它是使用者觸發的預定義工作流程。
